Videograph uses a flexible account model that supports organizations of any size, from individual developers to large OTT teams. This page explains how accounts, workspaces, and user roles work together to govern dashboard access and API behavior.

## Accounts

An account represents an individual user with access to the Videograph dashboard and API. Every user signs in with a personal account, which can belong to one or more workspaces.

### Key points

- Account identity is tied to authentication credentials.  
- Permissions are not assigned at the account level but through roles within each workspace.  
- Multiple accounts can collaborate in the same workspace.

## Workspaces and organizations

Workspaces (sometimes called organizations) are the primary container for your video operations. They hold all VOD assets, live streams, channels, metadata, SSAI configurations, API keys, and analytics.

### Workspace behavior

- Each workspace has its own API keys.  
- API requests are always scoped to a single workspace.  
- Dashboard visibility and editing rights depend on your role inside that workspace.

```json
{
  "workspace_id": "ws_12345",
  "name": "Production",
  "resources": {
    "assets": 128,
    "live_streams": 4,
    "channels": 2
  }
}
```

<Callout kind="tip">

Use separate workspaces for development, staging, and production when running complex pipelines.

</Callout>

## Roles and permissions

### Common role patterns

Below is a simplified example of typical permissions. Real permissions vary by workspace configuration, but this model captures the core pattern.

| Role | Create/update video assets | Manage live streams | Manage SSAI/ads | Manage workspace settings |
|------|-----------------------------|---------------------|------------------|----------------------------|
| Viewer | No | No | No | No |
| Editor | Yes | Yes | Limited | No |
| Admin | Yes | Yes | Yes | Yes |

## Recommended setups

Use these patterns as a starting point when organizing teams.

### Small teams

- One workspace for all environments.  
- One or two admins.  
- Everyone else as editors.

### Typical grouping

- Developers: editor access to development workspace.  
- Operators and media managers: editor access to staging and production.  
- Admins: full rights across all workspaces to configure SSAI, webhooks, and access policies.
